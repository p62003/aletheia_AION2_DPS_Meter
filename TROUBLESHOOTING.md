# 疑難排除指南

## 程式無法捕獲數據

### 快速診斷流程

```
程式能啟動嗎？
├─ 否 → 見「A. 程式無法啟動」
└─ 是
    浮窗 ms 有顯示數值嗎？
    ├─ 否（一直顯示 --ms）→ 見「B. 完全沒有封包」
    └─ 是（有 ms 數值）
        打怪後排行有任何反應嗎？（包含沒有名字的條目）
        ├─ 完全沒有任何反應 → 見「B2. 有延遲但無數據」
        └─ 有反應但不正確
            暱稱燈號是什麼顏色？
            ├─ 灰色 → 見「C. 暱稱自動偵測中」
            ├─ 橙色 → 見「D. 暱稱等待匹配」
            └─ 綠色 → 見「E. 封包正常但無傷害數據」
```

---

### A. 程式無法啟動

**症狀**: 雙擊後閃退、沒有視窗出現、報錯訊息

| 原因 | 解法 |
|------|------|
| **Npcap 未安裝** | 前往 https://npcap.com 下載安裝 Npcap（勾選 WinPcap API 相容模式） |
| **Npcap 版本過舊** | 解除安裝舊版，重新安裝最新版 Npcap |
| **缺少系統元件** | 安裝 Visual C++ Redistributable (x64) |
| **防毒軟體攔截** | 將程式加入防毒白名單（封包擷取工具容易被誤判） |

**排查方式**: 在程式所在資料夾開啟命令提示字元（cmd），輸入程式名稱執行，查看錯誤訊息。

---

### B. 完全沒有封包（ms 顯示 --ms）

**症狀**: 程式正常啟動，但延遲始終顯示 `--ms`，遊戲正常運行中

**原因**: 封包擷取層無法接收遊戲流量

| 原因 | 解法 |
|------|------|
| **未以系統管理員身分執行** | 右鍵 → 以系統管理員身分執行（封包擷取需要管理員權限） |
| **Npcap 安裝時未勾選相容模式** | 重新安裝 Npcap，勾選「Install Npcap in WinPcap API-compatible Mode」 |
| **使用 VPN/加速器** | 程式已自動相容大部分加速器。若仍有問題見「B2 步驟 3」 |
| **遊戲未連線** | 確認遊戲已經登入到角色選擇/進入遊戲（主選單/更新畫面沒有遊戲封包） |
| **防火牆阻擋** | Windows Defender 防火牆 → 進階設定 → 確認程式未被阻擋輸入規則 |
| **PPPoE 撥號連線** | 見「F. 已知限制 — PPPoE」 |

---

### B2. 有延遲但無數據

**症狀**: ms 有數值，但打怪後排行完全沒有任何反應，連無名條目都沒有

**這是什麼意思？**
ms 數值代表程式能看到你「送出去」的封包和伺服器的回應確認，但**不代表**程式能完整讀到伺服器回傳的遊戲數據。這兩者是分開的。

**依序嘗試以下步驟：**

**步驟 1 — 重新安裝 Npcap**
1. 完全關閉本程式和遊戲
2. 控制台 → 解除安裝 Npcap
3. 前往 https://npcap.com 下載最新版
4. 安裝時**務必勾選**：
   - ☑ Install Npcap in WinPcap API-compatible Mode
   - ☑ Support loopback traffic
5. 安裝完成後重新啟動電腦
6. 以系統管理員身分啟動本程式，再啟動遊戲測試

**步驟 2 — 關閉網路卸載功能**

Windows 的網路最佳化可能讓擷取工具看到不完整的封包：

1. 開啟「裝置管理員」
2. 展開「網路介面卡」→ 右鍵點擊你上網用的網卡 → 內容
3. 切到「進階」分頁
4. 找到並**停用**以下項目（不一定每項都有）：
   - Large Send Offload v2 (IPv4) → Disabled
   - Large Send Offload v2 (IPv6) → Disabled
   - TCP Checksum Offload (IPv4) → Disabled
   - Receive Segment Coalescing → Disabled
5. 確定 → 重新測試

> 停用後如果確認程式正常運作，可以之後逐一重新啟用，找出是哪一項造成衝突。
> 這些設定只影響效能最佳化，停用不會影響正常上網。

**步驟 3 — 遊戲加速器（ExitLag、GearUP、雷神、UU、迅游等）**

程式已內建通用加速器相容功能，會自動偵測加速器的代理連線並選擇正確的嗅探方式。大多數情況下**不需要額外設定**。

如果使用加速器時仍無法捕獲數據：

**方案 A — 確認啟動順序**：
1. 先啟動加速器並連線
2. 再啟動遊戲，進入角色畫面
3. 最後啟動本程式

> 程式啟動時會分析遊戲的網路連線，因此遊戲必須已經連線。

**方案 B — 使用診斷工具**：
1. 執行「Aletheia 網路診斷工具」（Aletheia_Diag.exe）
2. 依照畫面指引進行連線自檢
3. 在測試時間內攻擊怪物
4. 查看結果是否通過

> 如果自檢通過但主程式仍無法使用，可能是其他問題（見步驟 4、5）。
> 如果自檢也失敗，產生環境診斷報告後回傳給開發者。

**方案 C — 確認 Npcap Loopback 支援**：
- 大部分加速器透過 localhost 代理轉發遊戲流量
- 程式需要 Npcap 的 Loopback 嗅探功能來捕獲這些流量
- 重新安裝 Npcap 時，務必勾選「**Support loopback traffic**」

**方案 D — 暫時關閉加速器測試**：
1. 完全退出加速器
2. 直連遊戲，啟動本程式測試
3. 如果直連可以正常擷取 → 確認是加速器相容問題
4. 回傳診斷工具的環境報告給開發者

**步驟 4 — 確認網路介面**

如果電腦有多張網卡（例如同時有有線和無線），封包擷取工具可能抓錯介面：

1. 確認遊戲使用的是哪個連線（有線 / Wi-Fi）
2. 暫時停用不使用的網路介面卡：
   - 設定 → 網路和網際網路 → 進階網路設定
   - 停用未使用的介面卡
3. 重新啟動程式測試

**步驟 5 — 防毒/安全軟體**

某些防毒軟體會攔截或修改網路封包：

1. 暫時停用防毒軟體的「網路保護」或「封包檢查」功能
2. 將本程式加入防毒軟體的白名單/排除清單
3. 重新測試

**如果以上都無效**：請回報你的系統資訊（見底部「聯繫支援」），開發者需要進一步排查。

### C. 暱稱自動偵測中（灰色燈號 ●）

**症狀**: ms 有數值，但浮窗標題顯示預設文字，排名列空白

**原因**: 程式尚未從封包中偵測到你的遊戲暱稱

**解法**:
1. 確認遊戲已登入角色並進入遊戲世界
2. 進行一次場景切換（Loading）— 例如傳送、進入副本、切換頻道
3. 場景切換後對任意目標發動攻擊
4. 程式會自動從封包中擷取暱稱，燈號應在數秒內轉綠

> 暱稱偵測是全自動的，不需要手動輸入。程式透過遊戲封包（3336 opcode）識別你的角色。

---

### D. 暱稱等待匹配（橙色燈號 ●）

**症狀**: ms 有數值、暱稱已偵測到，燈號為橙色，排名列空白

**原因**: 已偵測到暱稱但尚未在封包中找到對應的 actor_id

| 情境 | 說明 |
|------|------|
| **剛進入場景** | 正常現象。攻擊或被攻擊後，程式即可從封包中配對你的 actor_id |
| **場景切換中** | 切換場景後 actor_id 會重新分配，需要再次攻擊以重新配對 |

**解法**: 對任意目標發動一次攻擊。燈號應在數秒內轉綠。

---

### E. 封包正常但無傷害數據（綠色燈號 ●）

**症狀**: ms 有數值、暱稱已匹配（綠色燈號），但攻擊目標後排名不動

| 原因 | 解法 |
|------|------|
| **全域模式需要職業識別** | 全域排行只顯示「有暱稱 + 有職業」的角色。剛開始時需要多打幾下讓程式偵測到你的職業 |
| **計時模式已結算** | 計時模式在最後一次攻擊後 10 秒自動結算（標題變紅+✓）。點重置按鈕 ↻ 開始新的計時 |
| **副本模式未觸發** | 進入副本後需要觸發第一次攻擊，程式才會切換到副本模式開始計算 |

---

### F. 已知限制

| 限制 | 說明 |
|------|------|
| **迅游加速器** | 迅游將遊戲流量導向本機實體 IP 的代理端口（非 localhost），這段流量不經過物理網線，Npcap 無法擷取。這是 Windows 網路架構的限制，非本程式的 bug。**建議**：改用其他已驗證的加速器（ExitLag、雷神、UU、GearUP、LagoFast） |
| **HiCn 海豚加速器** | HiCn 在 WFP/NDIS 驅動層攔截封包，Npcap 看不到遊戲流量。**建議**：改用其他已驗證的加速器 |
| **PPPoE 撥號連線（電腦端撥號）** | 詳見下方「**F-1. 電腦端 PPPoE 撥號**」專題說明 |
| **計時+BOSS 模式重疊** | 計時模式下進入 BOSS 戰，兩種模式可能同時運行 |

### F-1. 電腦端 PPPoE 撥號（無法捕獲數據）

**這是什麼？**

「PPPoE 撥號」是一種上網方式，常見於台灣中華電信、遠傳、台灣大寬頻的光世代/ADSL。
重點不是「有沒有用 PPPoE」，而是「**誰在做撥號這個動作**」：

| 撥號的位置 | 程式能用嗎？ | 說明 |
|---|---|---|
| **路由器撥號**（最常見） | ✅ 正常 | 數據機/光世代小烏龜接到無線分享器（華碩、TP-Link、小米…），由分享器登入帳密。電腦看到的就是普通的家用網路 |
| **電腦直接撥號**（少數） | ❌ 無法捕獲 | 數據機直接接電腦網路孔，每次開機要點桌面上「寬頻連線」輸入帳密才能上網 |

**為什麼電腦端撥號會失效？**

電腦自己撥號時，遊戲流量會走 Windows 內建的「**寬頻連線**」這個虛擬介面，而 Aletheia 使用的封包擷取套件（Npcap）**看不見**這個虛擬介面 — 這是 Windows 系統層面的限制，不是程式的 bug。同樣的情況下，如果改由路由器撥號，電腦看到的是普通的乙太網路流量，程式就能正常擷取。

---

**怎麼判斷自己是哪一種？**

最快的方法 — 看你**開機後需不需要手動點「寬頻連線」才能上網**：

- ✅ 開機後直接就能上網、開瀏覽器 → **路由器撥號**（不是這個問題，請看其他章節）
- ❌ 開機後要點桌面上「寬頻連線」圖示、輸入帳號密碼才能上網 → **電腦端撥號**（就是這個問題）

或者更明確的方法：
1. 按 `Win + R`，輸入 `ncpa.cpl`，按確定
2. 打開的視窗會列出你電腦上所有網路連線
3. 如果裡面有一個叫「**寬頻連線**」的項目，而且狀態是「已連線」 → 就是電腦端撥號

---

**解決方法 — 把撥號搬到路由器上**

需要準備：一台無線/有線分享器（路由器）。如果你已經有一台但目前只當 Wi-Fi AP 用，也可以重新設定它。

**簡易步驟**：

1. **準備帳號密碼**：找出你目前撥號用的 PPPoE 帳號和密碼（向你的網路業者申請的那組）。如果忘記了，可以打客服或在網路業者的會員網站查
2. **接線**：把光世代小烏龜（數據機）的網路線從電腦上拔下來，接到路由器的 **WAN 孔**（通常標示為 WAN 或顏色不同的孔）
3. **電腦接路由器**：用另一條網路線從路由器的 **LAN 孔**接到電腦（或改用 Wi-Fi 連到路由器）
4. **進入路由器設定頁**：開瀏覽器，網址列輸入 `192.168.1.1` 或 `192.168.0.1`（看路由器牌子，背面通常會印），輸入路由器管理員帳密
5. **找到 WAN/網際網路設定**：選擇連線類型為「**PPPoE**」，輸入步驟 1 的帳號密碼，儲存
6. **取消電腦上的「寬頻連線」**：`Win + R` → `ncpa.cpl` → 右鍵點「寬頻連線」→ 刪除（或停用）
7. **重開機測試**：電腦應該不需要任何手動動作就能直接上網。打開遊戲和 Aletheia，應該就正常了

> 💡 **不確定怎麼操作？** 直接打給你的網路業者客服說「我想把 PPPoE 撥號從電腦改到路由器」，他們會協助。或請懂網路的朋友幫忙 — 整個過程通常 10 分鐘內可以完成。

> ⚠️ **為什麼程式無法直接支援？** 技術上需要重寫封包擷取的核心架構來解析 PPPoE 封包。這是正在評估的未來改進方向，但需要時間驗證，目前先用上述方案處理。

### G. 其他常見問題

| 症狀 | 原因 | 解法 |
|------|------|------|
| **DPS 數值異常高** | 可能是初始幾秒內傷害集中導致 | 觀察 cDPS（戰鬥平均）而非即時 DPS |
| **浮窗拖不動** | 鎖定功能已啟用 | 點擊浮窗右上角的鎖定按鈕解鎖 |
| **浮窗位置在左上角** | 每次啟動程式會自動定位到遊戲視窗左上角 | 拖曳到喜歡的位置，同次執行期間會記住 |
| **切換語言後部分文字沒變** | 部分面板需要重新開啟 | 關閉並重新開啟浮窗或戰鬥細節面板 |

## 聯繫支援

如果以上步驟都無法解決問題，請提供以下資訊：

1. **診斷工具報告**（執行 Aletheia_Diag.exe → 環境診斷 → 桌面上的 JSON 檔案）
2. **作業系統版本**（Windows 10/11，64位元）
3. **Npcap 版本**（控制台 → 程式和功能 → 找 Npcap）
4. **網路連線方式**（有線 / Wi-Fi / 加速器名稱 / PPPoE 撥號）
5. **是否以系統管理員身分執行**
6. **浮窗的燈號顏色和 ms 數值**
